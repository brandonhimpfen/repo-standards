# Licensing

Licensing should not be an afterthought.

A repository without a license may be visible but not meaningfully reusable. Public access is not the same as permission.

## Guidance

Choose a license that fits the repository type and intent.

Examples:

- MIT for many software projects.
- Apache-2.0 when patent language is desired.
- CC0 or Creative Commons licenses for curated resources or data, where appropriate.

## Minimum Expectation

If a public repository is intended to be reused, adapted, or redistributed, a license should be included.
