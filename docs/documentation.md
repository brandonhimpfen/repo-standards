# Documentation

Documentation should reduce uncertainty.

## Minimum Documentation

At minimum, repositories should include a README that explains the project clearly enough for a new reader to understand what they are looking at.

## Strong Documentation Usually Includes

- Overview.
- Purpose.
- Scope.
- Usage or navigation guidance.
- Project status.
- Contribution guidance where relevant.

## Documentation Style

Good documentation is:

- Direct.
- Specific.
- Structured.
- Honest about limitations.

Overly promotional language should be avoided. Good documentation does not need hype. It needs clarity.
