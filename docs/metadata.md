# Metadata

Metadata improves discoverability and maintainability.

## Repository Platform Metadata

Where possible, set:

- Repository description.
- Homepage URL.
- Topics or tags.
- Social preview or project image where useful.

## Project Metadata

Where relevant, include:

- Author or maintainer.
- Package metadata.
- Keywords.
- Repository URL.
- Bugs or issues URL.
- Documentation URL.

Metadata should be consistent with the actual state of the project.
