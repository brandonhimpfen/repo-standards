# Contributing to Repo Standards

Thank you for your interest in improving Repo Standards.

## Scope of Contributions

Contributions are welcome when they improve clarity, strengthen the framework, fix errors, or add practical examples that align with the philosophy of the project.

Good contributions typically:

- Improve precision without adding unnecessary complexity.
- Strengthen the usability of the standard.
- Clarify tradeoffs or implementation guidance.
- Keep the framework broadly applicable.

## Before You Open a Pull Request

Please make sure the proposed change:

- Fits the purpose of the project.
- Preserves the lightweight and practical nature of the standard.
- Uses clear, direct language.
- Does not add tool-specific requirements unless clearly justified.

## Preferred Contribution Types

- Documentation improvements.
- Clarifications to the quality model.
- Better examples.
- Fixes for inconsistency or ambiguity.
- Small structural improvements.

## Pull Request Guidance

When submitting a pull request, please:

1. Explain what problem the change solves.
2. Keep the change focused.
3. Avoid bundling unrelated edits.
4. Update related documents where necessary.

## Philosophy

Repo Standards is intended to be durable, readable, and broadly reusable. Contributions should reinforce those goals.
